# 行为验证码500错误修复说明

## 问题原因

uni-app项目使用 `src/static` 目录作为静态资源目录，而不是 `public` 目录。

## 已修复内容

### 1. 文件迁移
已将TAC相关文件从 `public/` 移动到 `src/static/`：
```
src/static/
├── load.min.js          ✅ TAC加载器
├── captcha.html         ✅ 验证码HTML页面
└── tac/                 ✅ TAC资源目录
    ├── css/tac.css
    ├── js/tac.min.js
    └── images/
```

### 2. 更新路径引用

#### webview容器页面 (pages/common/captcha/index.vue)
```javascript
// H5平台
captchaUrl.value = `/static/captcha.html?...`;

// 小程序/APP平台
captchaUrl.value = `${baseUrl}/static/captcha.html?...`;
```

#### captcha.html
```javascript
// 动态获取basePath
const currentPath = window.location.pathname;
const basePath = currentPath.substring(0, currentPath.lastIndexOf('/'));

// 使用正确的TAC路径
const tacPath = basePath + '/tac';
const tac = await window.initTAC(tacPath, config, style);
```

## 注意事项

1. **static目录特性**
   - uni-app会自动将`src/static`目录的内容复制到输出目录
   - 访问时使用`/static/`前缀

2. **路径规则**
   - H5: `/static/xxx`
   - 小程序: `http://域名/static/xxx`
   - APP: `http://域名/static/xxx`

3. **不要删除public目录**
   - 保留作为备份
   - 将来可能用于其他用途
