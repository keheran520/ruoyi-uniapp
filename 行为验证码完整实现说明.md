# 行为验证码完整实现说明

## 概述

本项目已完整实现移动端行为验证码功能，支持所有uni-app平台：
- **H5平台**：直接使用TAC SDK
- **小程序/APP平台**：通过webview打开验证码页面

## 实现方案

### 1. H5平台
在H5浏览器中直接加载TAC SDK，无需跳转页面。

### 2. 小程序/APP平台
使用webview打开独立的验证码HTML页面，验证完成后通过`uni.postMessage`传递结果。

## 核心实现

### 1. Vite配置 (vite.config.js)

```javascript
export default defineConfig(() => {
    return {
        base: './',
        publicDir: 'public', // 指定public目录
        // ...
    }
})
```

### 2. 验证码HTML页面 (public/captcha.html)

独立的HTML页面，包含：
- TAC SDK初始化
- 验证成功/失败回调
- 通过`uni.postMessage`传递结果给uni-app

### 3. Webview容器页面 (pages/common/captcha/index.vue)

```vue
<web-view :src="captchaUrl" @message="handleMessage"></web-view>
```

监听来自webview的消息，将结果保存到`globalData`。

### 4. 工具类 (utils/behaviorCaptcha.ts)

使用条件编译区分平台：

```typescript
// H5平台
// #ifdef H5
  直接加载TAC SDK
// #endif

// 非H5平台  
// #ifndef H5
  打开webview页面
  轮询检查globalData中的结果
// #endif
```

### 5. 登录页面使用 (pages/login.vue)

```javascript
// 统一调用方式
const captchaId = await showBehaviorCaptchaModal({
  tenantId: loginForm.value.tenantId,
  apiBase: '/dev-api'
});

// 使用captchaId发送验证码
await sendEmailVerifyCode(email, tenantId, captchaId);
```

在`App.vue`中：


### 3. 配置Public目录

确保`vite.config.js`中配置了`publicDir: 'public'`。

## 使用示例

### 发送邮箱验证码

```javascript
async function sendEmailCode() {
  // 验证邮箱格式
  if (!emailRegex.test(loginForm.value.email)) {
    modal.msgError("请输入正确的邮箱地址");
    return;
  }
  
  try {
    // 显示验证码（自动适配平台）
    const captchaId = await showBehaviorCaptchaModal({
      tenantId: loginForm.value.tenantId,
      apiBase: '/dev-api'
    });
    
    // 发送邮箱验证码
    modal.loading("发送中...");
    const res = await sendEmailVerifyCode(
      loginForm.value.email, 
      loginForm.value.tenantId, 
      captchaId
    );
    
    if (res.code === 200) {
      modal.msgSuccess("验证码已发送");
      startEmailCountdown();
    }
  } catch (error) {
    console.log('用户取消或验证失败');
  }
}
```

## 数据流程

### H5平台

```
用户点击 → loadTACScript() → 显示TAC弹窗 
→ 用户滑动验证 → TAC回调 → resolve(captchaId)
```

### 小程序/APP平台

```
用户点击 → navigateTo(captcha页面) → webview加载captcha.html
→ 用户滑动验证 → uni.postMessage → globalData.captchaResult
→ 轮询检测到结果 → resolve(captchaId) → navigateBack
```

## API接口

所有平台使用相同的后端接口：

### 生成验证码
```
POST /dev-api/captcha/gen
Body: { type: 'SLIDER' }
```

### 校验验证码
```
POST /dev-api/captcha/check
Body: { id: string, data: string }
```

### 发送邮箱验证码
```
GET /resource/email/code
Params: { email, businessType, captchaId, tenantId }
```

### 发送手机验证码
```
GET /resource/phoneverify/code  
Params: { phonenumber, businessType, captchaId, tenantId }
```

## 调试说明

### H5平台调试

1. 确保`public/load.min.js`和`public/tac/`存在
2. 检查浏览器Console是否有加载错误
3. 查看Network标签页TAC资源是否正确加载

### 小程序/APP平台调试

1. 确保`public/captcha.html`存在
2. 在webview的`@message`事件中打印消息内容
3. 检查`getApp().globalData.captchaResult`是否正确设置

## 常见问题

### Q1: H5平台404错误
**A:** 检查`vite.config.js`是否配置了`publicDir: 'public'`，重启开发服务器。

### Q2: 小程序webview白屏
**A:** 确认小程序业务域名配置，添加验证码服务器域名到白名单。

### Q3: 验证码无法显示
**A:** 检查`/dev-api/captcha/gen`接口是否正常，是否返回了图片数据。

### Q4: 验证成功但没反应
**A:** 检查`globalData.captchaResult`是否正确设置，检查轮询逻辑是否执行。

## 注意事项

1. **跨域问题**：webview中的请求需要配置CORS或代理
2. **域名白名单**：小程序需要配置业务域名
3. **HTTPS要求**：生产环境webview必须使用HTTPS
4. **超时处理**：设置了30秒超时，可根据需要调整
5. **内存清理**：验证完成后及时清空`globalData.captchaResult`
